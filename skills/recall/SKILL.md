# Recall

在你系統提示的「# auto memory」段落裡已經給出這個專案的記憶目錄路徑與 MEMORY.md 索引格式。這個 skill 只負責「使用者主動要求查詢」時的搜尋與呈現，不重新定義格式。

如果目前 session 的系統提示裡沒有「# auto memory」段落，告知使用者這個專案還沒有自動記憶系統可查，並停止。

## 流程

1. 先看 MEMORY.md 索引。`remember` skill 會把索引依主題用 `## <topic>` 分節，先看有沒有語意相符的 topic 節（不用完全字面match），再看該節底下的條目；也用 grep 找關鍵字比對整份索引，兩者互補。
2. 索引比對到的候選，如果使用者的問題需要細節，再讀對應的記憶檔案取得完整內容；不要把所有記憶檔案都讀一遍。
3. 找不到相符內容：直接說「目前記憶裡沒有關於 XX 的紀錄」，不要用其他知識腦補一個答案充數。
4. 找到的記憶如果涉及「現在還存不存在」這種可能過時的具體事實（檔案路徑、函式名、某個決定是否仍然有效），依系統提示「Before recommending from memory」的規則，先用現況核對過再引用，不要直接把記憶內容當作現在的事實丟給使用者。
